# Security Policy

## Reporting a Vulnerability

We take the security of The Composable Co-Developer seriously.

If you discover a security vulnerability, please report it privately by opening a security advisory on GitHub:

1. Go to the [Security tab](https://github.com/ahmiershadowman-commits/the-composable-co-developer/security)
2. Click "Report a vulnerability"
3. Provide details about the vulnerability

## What to Include

Please include the following information in your report:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if any)

## Response Time

We aim to respond to security reports within 48 hours.

## Security Best Practices

When using this marketplace:

1. **Review plugin code** before installation
2. **Check permissions** requested by plugins
3. **Use trusted sources** for additional plugins
4. **Keep dependencies updated**
5. **Run in appropriate permission mode** for your use case

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | ✅ Yes    |

## Security Updates

Security updates will be released as patch versions (e.g., 0.1.1 → 0.1.2).

Subscribe to releases to stay informed about security updates.
