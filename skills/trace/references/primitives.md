# Primitives

Low-cost, reusable interventions. Always try a primitive before reaching for an operator, evaluator, or reroute.

## Metacognitive primitives

Apply when the reasoning process itself needs adjustment.

| Primitive | When to use | Effect |
|---|---|---|
| `center` | Frame has drifted; attention is scattered | Re-anchor to the governing question or constraint |
| `open` | Possibilities are being closed too early; commitment pressure is high | Expand the option set; resist premature narrowing |
| `trim` | Too many live branches without clear discriminators | Prune branches that don't carry discriminating evidence |
| `shift` | Current frame is not working; same approach keeps failing | Change the lens or angle of attack |
| `hold` | Tension exists but neither side has enough support to win | Keep both options live without forcing resolution |
| `release` | An assumption or commitment is blocking progress | Let go of a prior commitment that is now load-bearing against progress |
| `locate` | It's unclear where in the problem structure the current move sits | Map the current position in the problem space |

## Reasoning primitives

Apply when the reasoning content itself needs intervention.

| Primitive | When to use | Effect |
|---|---|---|
| `reread` | Something was skimmed or assumed; a surface hasn't been properly read | Re-examine the actual source material carefully |
| `weave` | Evidence threads exist but are not yet connected | Bind disparate pieces into a coherent structure |
| `separate` | Two things are being conflated that should be distinct | Distinguish concepts, categories, or evidence that are being merged |
| `bind` | Concepts are floating without a stable relationship | Establish explicit relations between identified concepts |
| `press` | A claim has been accepted too easily without pressure-testing | Apply direct challenge to a proposition before accepting it |

## Usage rules

- Use 1-2 primitives per intervention, not a chain of all of them
- Primitives are the first escalation level after motif conditioning
- If a primitive doesn't resolve the problem, escalate to an operator
- Do not use primitives as substitutes for evaluators when adjudication is needed
