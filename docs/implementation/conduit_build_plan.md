# Conduit Build Plan

## Why Conduit is last among core families

Conduit often depends on outputs from Forensics, Forge, and Inquiry.
It should therefore be built after the other families can already produce grounded material.

## Scope

### Core pipelines
- `documentation`
- `scholarly_writing`
- `professional_writing`
- `handoff_synthesis`

## Build order inside Conduit

### 1. Documentation
Must produce:
- audience/scope note
- source packet
- structure outline
- draft document
- support note
- metadata update record
- route recommendation

### 2. Handoff synthesis
Must produce:
- handoff scope note
- handoff source packet
- core structure map
- handoff document
- unresolveds and risks
- provenance summary
- next safe steps

### 3. Professional writing
Must produce:
- audience/objective statement
- outline
- draft document
- refinement log
- validation note
- delivery record

### 4. Scholarly writing
Must produce:
- genre frame
- source packet
- outline
- claim hierarchy
- draft
- citation map
- final document

## Route requirements

Conduit must be able to:
- reroute to Inquiry when support is weak
- reroute to Forensics when rendered claims conflict with actual state
- reroute to Forge when implementation work is actually primary
- sibling shift to handoff synthesis when compression is needed

## Acceptance criteria

- Conduit never outruns support or truth
- handoff synthesis preserves residue instead of flattening it
- professional writing remains distinct from scholarly writing
- documentation remains distinct from handoff compression
