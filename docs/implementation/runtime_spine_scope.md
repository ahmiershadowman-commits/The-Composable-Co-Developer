# Runtime Spine Scope

## What the spine is

The runtime spine is the minimum live kernel that enables:
- Loading and indexing the marketplace spec bundle
- Validating schema and inventory consistency
- Resolving canonical targets to typed objects
- Executing Trace as the traveling metacognitive controller
- Dispatching Residue lenses for suspicious-surface investigation
- Escalating to Lever for bounded evaluation
- Transitioning legally across phase, pipeline, and family boundaries
- Emitting artifacts and provenance records
- Running one complete vertical slice end-to-end

## What the spine is not

The spine is **not**:
- A full MCP implementation
- A complete hook system
- A user interface
- An experimental pipeline executor
- A full adapter surface
- Family-complete before proven

## What is deferred

The following are explicitly out of scope for the spine phase:

1. **MCP implementation** - Model Context Protocol integration deferred
2. **Hook system** - PreToolUse, PostToolUse, etc. deferred
3. **UI/visualization** - No graphical interface
4. **Experimental pipelines** - Only core pipelines in vertical slice
5. **Full family completeness** - Spine proves architecture first
6. **Persistence layer** - In-memory state for spine validation
7. **Concurrent execution** - Single-threaded for simplicity
8. **External integrations** - No API calls beyond local file system

## Spine success criteria

The spine is complete when:
1. Registry can load and index entire bundle
2. Every runtime target resolves to typed object or structured error
3. Trace can choose smallest-sufficient action
4. Residue can propose structured interventions
5. Lever can return structured evaluation results
6. Transitions move legally between phases/pipelines/families
7. Artifacts and provenance are written to disk
8. Vertical slice runs: Forensics → Defragmentation → Forensics → Forge

## Post-spine work

After spine validation:
- Implement full family pipelines
- Add MCP integration
- Build hook system
- Add experimental pipeline support
- Implement persistence
- Add concurrency support
