# Dependency Graph Summary

For quick reference, the primary dependency law is:

**Forensics before trust-dependent work.  
Trace before escalation.  
Lever after Trace.  
Registry before resolution.  
Resolution before transitions.  
Transitions before artifacts.  
Artifacts before handoff.  
Conduit after support.**

The implementation order this implies is:

1. schemas + target grammar
2. registry
3. target resolver
4. Trace
5. Residue
6. Lever
7. transitions
8. artifacts/provenance
9. vertical slice
10. Forensics
